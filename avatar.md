Avatar 类解析
==========

Avatar 类是游戏中的核心角色类，继承自 AvatarBase，包含了玩家和NPC的所有属性和行为逻辑。以下是详细解析：
主要功能
----

### 1. 基础属性

* ​**角色属性**​：HP、MP、攻击、防御、闪避等战斗属性
* ​**五行属性**​：金木水火土五行值
* ​**灵根系统**​：灵根类型和数值
* ​**境界系统**​：角色等级和境界
* ​**寿元系统**​：年龄和寿元限制

### 2. 战斗系统

* ​**技能管理**​：普通技能、静态技能(功法)、结丹技能
* ​**灵气系统**​：灵气类型、获取和使用
* ​**战斗状态**​：回合制战斗状态管理
* ​**Buff/Debuff**​：各种状态效果管理

### 3. 物品系统

* ​**装备管理**​：武器、防具、饰品等
* ​**物品背包**​：物品获取、使用和丢弃
* ​**丹药系统**​：丹药效果和使用

### 4. 社交系统

* ​**人际关系**​：师徒、兄弟、道侣等关系
* ​**门派系统**​：门派归属和声望
* ​**传音符系统**​：与其他角色通信

### 5. 时间系统

* ​**年龄增长**​：随时间变化的属性
* ​**时间事件**​：特定时间触发的事件

核心组件
----

### 1. 子系统管理

* `CardMag cardMag` - 灵气卡牌管理
* `Combat combat` - 战斗系统
* `Spell spell` - 法术系统
* `BuffMag buffmag` - Buff管理
* `WuDaoMag wuDaoMag` - 悟道系统
* `WorldTime worldTimeMag` - 游戏时间管理

### 2. 数据存储

* `StreamData StreamData` - 玩家存档数据
* 各种JSONObject字段存储游戏状态

### 3. 战斗相关

* `FightTempValue fightTemp` - 战斗临时数据
* `List<GUIPackage.Skill> skill` - 当前装备的技能
* `List<StaticSkill> StaticSkill` - 静态技能(功法)

重要方法
----

### 1. 属性相关

* `setHP()` - 设置生命值，处理伤害和死亡
* `levelUp()` - 角色升级逻辑
* `addEXP()` - 经验值获取

### 2. 战斗相关

* `recvDamage()` - 接收伤害处理
* `useTargetSkill()` - 使用目标技能
* `addCrystal()` - 获取灵气
* `UseCryStal()` - 使用灵气

### 3. 物品相关

* `addItem()` - 添加物品到背包
* `removeItem()` - 移除物品
* `equipItem()` - 装备物品
* `YSequipItem()` - 特殊装备逻辑

### 4. 技能相关

* `equipSkill()` - 装备技能
* `equipStaticSkill()` - 装备功法
* `addSkill()` - 添加技能到战斗列表

### 5. 系统功能

* `Load()`/`Save()` - 存档读档
* `onEnterWorld()` - 进入世界时的初始化
* `die()` - 角色死亡处理

特殊系统
----

### 1. 突破系统

* 筑基、结丹、结婴、化神等突破逻辑
* 特殊突破战斗处理

### 2. 天劫系统

* 飞升渡劫相关逻辑
* 天劫技能和效果

### 3. 锻体系统

* 特殊锻体功法和效果

### 4. 悟道系统

* 悟道点和悟道技能

数据存储结构
------

Avatar 类使用了大量JSONObject来存储游戏数据，包括：

* 任务数据
* 物品数据
* 技能数据
* 社交关系
* 门派好感
* 成就系统

注意事项
----

1. 该类同时处理玩家和NPC逻辑，通过`isPlayer()`区分
2. 大量使用事件系统(`Event.fireOut`)进行模块间通信
3. 与UI系统有紧密交互，特别是战斗UI
4. 包含大量过时方法(`[Obsolete]`)，新代码应避免使用

这个类是游戏中最核心的类之一，几乎涉及游戏的所有核心系统，代码量巨大且功能复杂。
